Metodologías

Para realizar la práctica 5 orientada al diseño de esta página web lo primero que se ha hecho es descargar los ejemplos de Bootstrap, una biblioteca de ejemplos de plantillas que permite la creación de la página web. Posteriormente, se ha usado 7-zip para hacer unzip de sticky-footer-navbar. Del mismo modo, se realizó un unzip para descomprimir la carpeta de “assets”, que contenía “dist” en su interior, y esta a su vez “css”. Esta última carpeta es la verdaderamente interesante, ya que contiene el estilo de Bootstrap.min. Una vez descomprimidos los archivos, se ha procedido a crear diversas carpetas con el comando mkdir. La primera de ellas, docs, contiene a su vez la carpeta css y la carpeta js, que llevará dentro el archivo Bootstrap.bundle.min.js. Después, se ha utilizado el comando mv para mover los archivos a las nuevas carpetas. Así, se han hecho los siguientes cambios:

  • Los contenidos de sticky-footer-navbar, index.html y sticky-footer-header.css se han movido a docs.
  • Bootstrap.min.css se ha movido a css y css se ha movido a docs.
  • Bootstrap.bundle.min.js se ha movido a js y js se ha movido a docs. Una vez creada la carpeta docs, desde el repositorio de GitHub, se ha accedido a Settings > Pages y se ha seleccionado como rama principal docs. Después se ha guardado. Seguidamente se han creado nuevos archivos que dan forma a la página web dentro de la carpeta docs (en formato html). Estos son:
  • Cabecera.html
  • Pie.html
  • Práctica1.html
  • Práctica2.html
  • Práctica3.html
  • Metodologías.html

    En ellos, se ha copiado el formato de index.html y se ha pegado el contenido de las prácticas (o se ha escrito un texto en aquellos como metodologías) en la parte correspondiente, debajo de “Begin page content”. Sin embargo, para pegar el contenido de las prácticas hay que utilizar lenguaje html. Para conseguir transformar este lenguaje markdown en html se ha utilizado Pandoc. En la terminal, se ha hecho apt-cyg-install Pandoc. Una vez convertidos a html 5, se han pegado en los nuevos archivos. Asimismo, se ha editado el Header para lograr que apareciesen todas las prácticas accesibles en la cabecera, y que fuese posible realizar un retorno a la página de inicio situada en el Index.html.
